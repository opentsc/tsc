# TSC — 魂 · 薄壳公司白皮书

**The Thin-Shell Company — Laws of a Self-Evolving Organization**

一部「法」，不是说明书。它定义一个组织"为什么存在、什么算好、怎么进化"——同时被人读懂、被 AI 执行。

> **TSC 是魂，OpenTSC 是壳。**
> 魂（本仓）= 这套法 + 组织积累的全部判断与记忆。抽象、可移植，决定"为什么"和"什么算好"。
> 壳（[`opentsc/opentsc`](https://github.com/opentsc/opentsc)）= 运行这套法的容器。
> 薄壳公司的全部赌注是"一个人驱动一个大组织"。壳越薄，魂必须越厚、越能脱离任何具体的人和软件而存活。

## 总纲：魂与壳

```
       魂 (Soul) — 可移植、跨壳存活
   ┌──────────────────────────────────┐
   │  创世层      不可变的存在承诺        │
   │  判断法典    什么算好/可信/值得动     │
   │  事件流      带证据的全部轨迹         │
   │  校准记忆    判断对过错过的账本        │
   └──────────────┬───────────────────┘
                  ↓ 注入
           壳 (Shell) — 可替换的运行容器
```

魂能从一个壳里完整倒出来，注进另一个壳里复活。软件可重写、团队可换人、模型可升级——只要魂还在，这个 TSC 就还是同一个 TSC。这是忒修斯之船问题在组织层面的答案。
