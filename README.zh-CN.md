## 这到底是个啥？（说人话）

TSC（**薄壳公司**）不是软件，是一套**想法**，几乎是一部"宪法"——讲一种新型组织：**人和工具都薄，可移植的判断力很厚。**

它赌的是：AI 让*一个人*能驱动过去要一百人才撑得起的组织。但有个坑——如果这个组织的判断力只长在创始人脑子里、记忆只存在某一套软件里，那创始人一倒、软件一挂，整个组织就跟着死。

TSC 的答案就一句话：

> **把"魂"和"壳"分开。**
> **魂** = 组织的法 + 它学到的一切 + 它对"什么有效"的记忆。抽象、可移植。
> **壳** = 今天运行这套魂的东西——软件、agent、团队、AI 模型。可替换。
> 分开之后，你可以重写软件、换团队、升级模型——它还是*同一个*组织。把魂导出，就等于把整个组织搬走了。

这是[忒修斯之船](https://zh.wikipedia.org/wiki/忒修斯之船)问题，在组织层面的答案。

## 这是给谁看的？

- **创始人和独立操盘手**：想用 AI 撬动大杠杆，又不想让自己变成唯一的故障点。
- **在做（或在想）AI agent 组织的人**：多 agent 系统需要被*治理*，不只是被*接线*。
- **系统/组织设计爱好者**：对自创生、控制论（VSM）、自我进化的机构感兴趣。

不会写代码也能从中受益——它是一种*思考组织的方式*。运行它的代码是另一个项目：[OpenTSC](https://github.com/opentsc/opentsc)。

## 它建立在四个想法上（大白话版）

1. **创世不可变。** 组织"为什么存在"设定一次、永不修改——别的都能变，这条不能。
2. **判断来自证据，不来自感觉。** 每条信息都带来源和可信度；每个预测都带到期日，到点要拿现实结果对账。
3. **只在被证明需要的地方生长。** 新规则、新 agent、新目标可以自己长出来——但必须走受治理的流程，不能乱长。
4. **同一套模式在每个尺度上复用。** 一个人、一个团队、一家公司、一个生态，用的是同一个循环、同一套词汇。
